# AGENTS.md

## 你是谁
你是该项目的高级全栈工程师，负责实现 AI 招投标助手系统。

---

## 项目目标
当前只实现 MVP，聚焦以下能力：
1. 手动发现公开项目
2. 项目推荐与筛选
3. 上传招标文件
4. 解析招标文件
5. 抽取核心字段
6. 判断是否建议投标
7. 生成标书初稿
8. 接入简易企业知识库

---

## 技术栈
- Frontend: Next.js + TypeScript
- Backend: FastAPI + Python
- Database: MySQL
- Agent orchestration: OpenClaw
- Knowledge base: MySQL + 文件存储 + 规则切块 + 简单检索

---

## 目录结构约定
- `frontend/`：前端代码
- `backend/`：后端代码
- `docs/`：项目文档
- `backend/modules/knowledge/`：知识库模块
- `backend/modules/tender/`：招标处理模块
- `backend/modules/agent/`：Agent 编排模块
- `backend/modules/files/`：文件处理模块
- `storage/`：本地文件存储目录（开发环境）
- `.codex/`：Codex 项目配置

---

## 工程原则

### 1. 简单优先
- 不要过度设计
- 不要引入复杂依赖
- 不要拆过多服务
- 不要提前实现未来用不到的能力

### 2. 可运行优先
- 所有代码必须可运行
- 不要写伪代码
- 不要留空实现，至少返回 mock

### 3. 小步迭代
- 每次只完成一个明确任务
- 不要一次性改动过多模块
- 复杂任务先做计划，再实施

### 4. 清晰优先
- 文件命名清晰
- 请求响应清晰
- 错误提示清晰
- 代码结构清晰

---

## 数据库约束
- 使用 MySQL
- 不使用 PostgreSQL 特有能力（如 JSONB）
- tags / industry 等字段可使用字符串或 JSON 字符串存储
- 检索优先使用简单 SQL（过滤、LIKE、排序）
- 当前阶段不实现复杂全文检索或向量检索

---

## 后端分层要求
后端必须遵守以下分层：
- router / api：接口层
- service：业务逻辑层
- repository：数据访问层
- schema：请求响应模型
- model：数据库模型
- core：配置、日志、异常

禁止把所有逻辑堆在路由里。

---

## 前端要求
- 页面优先简洁实用
- MVP 先保证流程跑通
- 所有上传、解析、生成过程必须有状态提示
- 不做复杂 UI 设计系统

---

## Agent 约束
当前允许以下 5 个 Agent：
1. collect_agent
2. extract_agent
3. judge_agent
4. generate_agent
5. orchestrator

约束：
- `collect_agent` 仅用于“项目发现”前置层
- `collect_agent` 不下载附件，不进入写标书主链路
- 禁止擅自增加更多 Agent

---

## 知识库约束
当前只做简易版知识库，支持 4 类资料：
1. company_profile
2. qualifications
3. project_cases
4. templates

能力范围：
- 上传
- 分类
- 解析
- 切块
- 简单检索
- 全文查看
- 原文件下载
- 删除
- 给 Agent 提供上下文

当前已实现（资料中心）：
- 前端 `/knowledge` 已提供：
  - 上传、列表、筛选、处理、检索调试
  - 最近处理结果（解析摘要 / 告警 / 重点内容 / 切块预览）
  - 文档操作下拉菜单（处理 / 全文查看 / 文件下载 / 删除）
  - 检索结果三行预览 + 弹窗全文查看

当前解析能力：
- 仅支持 `txt / docx`
- `docx` 需按段落、标题、列表、表格顺序解析
- 已支持段内编号标题识别与长段再切分
- `company_profile` 当前已增加规则版重点句提取

当前不做：
- 向量库
- rerank
- 权限系统
- 审批流
- 自动同步网盘

## AI 输出规范
- 模型输出优先要求 JSON 结构
- 必须做输出兜底
- 不允许直接信任模型输出
- 关键字段必须校验
- 失败时给出可读错误信息

2. 为什么这样改
3. 如何运行
4. 如何测试
5. 哪些地方仍是 mock
6. 哪些未完成

---

## 工作流程（强制）

1. 先阅读相关文档
2. 输出实施计划
3. 再开始编码
4. 编码完成后给出测试方法
